# Agent3 接口详情获取逻辑修正说明

## 修正目标

统一 Agent3 的职责边界，确保：

1. 知识库主文件只保存基础接口索引。
2. Agent3 只能先用基础索引定位候选接口。
3. Agent3 只能对本次命中的接口一对一获取详情。
4. Agent3 不允许全量拉取整个项目的所有接口详情。

## 当前规则

- 知识库主索引用于定位接口，不用于保存完整接口契约。
- 详情来源仅限 Apifox、接口文档或代码实现。
- 详情获取范围仅限本次需求涉及的接口集合。
- 无法确认的字段必须标记为“待确认”。

## 本次修正内容

- 调整 Orchestrator：
  - 当输入是 Apifox 基础知识库索引时，先保留索引用于定位；
  - 再根据测试用例筛选本次命中的接口；
  - 最后只对这些接口调用按接口范围的详情获取。
- 调整 ApiDocumentResolver：
  - 新增项目基础索引解析能力；
  - 保留按接口集合获取详情的能力；
  - 不再把整项目完整 OpenAPI 作为 Agent3 主流程默认输入。
- 调整 ApiMapperAgent：
  - 新增基于基础索引的候选接口筛选逻辑；
  - 明确详情状态与“待确认”标记规则；
  - 明确知识库只负责定位，详情必须按需获取。

## 结果

- Agent3 主流程不再依赖整项目全量详情获取。
- Agent3 输出的完整契约仅覆盖本次命中的接口。
- 知识库主索引与 Agent3 详情获取职责分层更清晰。
